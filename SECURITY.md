# Security Policy

Report security issues privately to `architects@zer0pa.ai`.

Do not open a public issue for:

- secrets or tokens committed to the repo
- dependency or supply-chain vulnerabilities
- arbitrary-code-execution or privilege-escalation bugs

Engineering disagreements, falsification findings, and claim-boundary disputes
are not security reports. Those belong in normal issue or review flow.

Current staged versions supported for fixes:

- `0.0.0` private staging snapshot
